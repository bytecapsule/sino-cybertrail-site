西班牙殖民者早期征服美洲时绑架原住民国家元首勒索财物，与2026年特朗普政府绑架委内瑞拉总统马杜罗并企图控制其石油资源的行为，本质均为强国对弱国的资源掠夺，二者在行为逻辑、权力压迫本质上具有高度相似性，均是霸权主义在不同历史时期的具体表现。委美冲突

16世纪西班牙殖民者征服美洲时，通过欺骗、突袭等手段绑架原住民国家元首，以生命安全或政权存续为要挟，勒索大量财富（黄金、白银等），并将其作为殖民扩张的核心资金来源。这种行为贯穿了西班牙对美洲的征服过程。

1519年，西班牙征服者埃尔南·科尔特斯利用阿兹特克人对“白皮肤神明”的迷信，以“和平会晤”为名进入北美阿兹特克帝国首都特诺奇蒂特兰，绑架了阿兹特克皇帝蒙特祖玛二世。随后，科尔特斯以蒙特祖玛的生命为要挟，勒索了巨额的黄金、白银和珠宝（包括阿兹特克人的圣物“太阳石”），并将这些财富运回西班牙。即使蒙特祖玛后来被同胞处决，西班牙仍通过其继任者夸特莫克继续勒索，直至彻底摧毁阿兹特克帝国。

1532年，弗朗西斯科·皮萨罗利用印加帝国因内战陷入混乱的机会，以“谈判”为名诱骗印加皇帝阿塔瓦尔帕进入卡哈马卡广场，随后发动突袭，绑架了阿塔瓦尔帕。皮萨罗要求印加人用黄金填满一间长22英尺、宽17英尺的房间（史称“赎金房间”），以换取阿塔瓦尔帕的自由。印加人耗尽全国黄金（约13265磅）和白银（约26000磅）满足其要求后，皮萨罗却背信弃义，将阿塔瓦尔帕绞死，随后洗劫了印加帝国的首都库斯科，掠夺了大量财富。

这些行为并非孤立事件，而是西班牙殖民体系的核心策略：通过绑架原住民首领，削弱其政权的合法性，同时获取大量财富，为后续的殖民统治奠定基础。正如历史学家所指出的，“西班牙的征服不仅是军事上的胜利，更是通过对原住民首领的控制，实现了对资源和财富的掠夺”。

2026年1月3日，特朗普政府以“打击毒品恐怖主义”为借口，对委内瑞拉发动军事突袭，绑架了总统尼古拉斯·马杜罗及其夫人，并将其带至美国受审。尽管美方声称行动是“执法行为”，但真实目的是控制委内瑞拉的石油资源。

1月7日，特朗普在个人社交媒体账号宣布，直接向委内瑞拉掠夺3000万至5000万桶“高质量、受制裁的石油”，将由储油船运送，运往美国的卸货码头并出售，所得资金将由他本人管控。这批原油的售价将高达30亿美元。正如中国国际问题研究院研究员王友明所言，“美国的经济动机已摆在台面上，控制委内瑞拉石油资源是其核心目标”。

特朗普在记者会上明确表示，“美国将从委内瑞拉的石油中获得补偿”，并计划让美国大型石油公司（如美孚、雪佛龙）进入委内瑞拉，修复其破败的石油基础设施，将石油资源“变现”为美国的财富。这种行为与西班牙殖民者掠夺黄金、白银的逻辑完全一致，都是通过控制弱国的资源，为强国谋取经济利益。

西班牙殖民者的“绑架-勒索”与特朗普政府的“绑架-资源控制”，尽管相隔500年，但本质逻辑高度一致，均体现了强国对弱国的资源掠夺和霸权主义，具体相似性如下：

• 行为模式一致：均采用“欺骗/突袭+绑架”的手段，针对弱国的国家元首（阿兹特克皇帝、印加皇帝、委内瑞拉总统），以“生命安全”或“政权存续”为要挟，实现资源掠夺（黄金、白银/石油）的目的。

• 权力压迫本质一致：均是强国利用其军事、经济优势，对弱国实施不对称压迫。西班牙殖民者通过枪炮、马匹等优势，击败原住民的石制武器；特朗普政府则通过军事突袭、经济制裁等手段，压制委内瑞拉的反抗能力。二者均无视弱国的主权，将弱国视为“资源供给地”。

• 目的高度重合：均以获取资源为核心目标。西班牙殖民者需要黄金、白银来支撑其国内的资本主义发展；特朗普政府则需要委内瑞拉的石油来巩固其能源霸权，降低国内油价，同时为美国石油公司谋取巨额利润。

从16世纪西班牙殖民者的“黄金掠夺”到21世纪特朗普政府的“石油控制”，霸权主义的本质从未改变——强国通过军事、经济手段，掠夺弱国的资源，维护自身的全球主导地位。这种行为不仅给弱国带来了巨大的灾难（如美洲原住民的灭绝、委内瑞拉的经济崩溃），也破坏了国际秩序的公平与正义。

正如联合国秘书长古特雷斯所言，“美国的行动开创了危险的先例，违反了《联合国宪章》和国际法”。历史已经证明，霸权主义终将遭到反噬，任何试图通过掠夺弱国资源来维持霸权的行为，都将付出沉重的代价。

总之，西班牙殖民者的“绑架-勒索”与特朗普政府的“绑架-资源控制”，是霸权主义在不同历史时期的具体表现，其本质均是强国对弱国的资源掠夺，这种逻辑不仅不符合现代国际秩序的基本准则，也必将遭到国际社会的强烈谴责。

对《民族团结进步促进法（草案）》的深度审视

长期以来，我国在民族问题领域的政策与法规实践，逐渐显露出一种值得深入反思的结构性偏差。在政策推行过程中，过度聚焦于少数民族的特殊性并给予政策倾斜，却对主体民族文化在凝聚中华民族共同体中所发挥的基础性作用予以系统性忽视。

从人口结构层面来看，汉族人口约占全国总人口的90%，然而在制度构建与话语表达体系中，汉族却常常被简单化地视为“56个民族中的一个”。民族政策的讨论往往围绕“差异保护”“特殊照顾”等议题展开，却鲜少正面回应一个根本性的       问题：作为中华文明历史的主要承载者，主体民族文化如何能够成为多民族国家公共文化与制度整合的坚实基础？

更为值得警惕的是，在众多政策与法律表述中，“民族团结”“民族平等”并未稳固地扎根于宪法所确立的人民个体一律平等原则之上，而是逐渐异化为一种以“民族身份”作为基本治理单元的逻辑。这种逻辑一旦固化，权利与义务的探讨便极易从“人”的层面滑向“族别”的范畴，进而动摇宪法意义上人人平等原则的根基。

在此背景下，对《中华人民共和国民族团结进步促进法（草案）》的审视，其意义已远超文本技术层面，而是关乎中国民族治理理念能否实现自洽的关键所在。

一、共同体与自治并置：被刻意回避的制度矛盾

草案中反复强调“铸牢中华民族共同体意识”，同时重申“坚持和完善民族区域自治制度”。然而，二者之间存在的内在张力，在草案中却被刻意淡化处理。

从制度逻辑的角度分析，中华民族共同体理念强调的是融合、嵌入以及共同身份的形成，其目标在于弱化族群边界、强化整体认同；而民族区域自治制度则以族群差异为前提，通过制度化的手段确认并固化这种差异。

回顾历史，民族区域自治作为一种制度安排，有着特定的时代背景和阶段性合理性。它诞生于国家整合的初期阶段，主要目的在于安抚差异、稳定边疆、实现国家统一。但当国家治理目标明确转向“深度融合”和“全方位嵌入”时，继续强化以民族身份为核心的自治逻辑，必然会在理论上出现不自洽的情况。

换句话说，一边大力倡导“去边界化”的共同体建设，一边又在制度层面不断确认和强化族群边界，这本身就是一种内在的矛盾。草案对这一矛盾并未作出回应，而是选择将两种逻辑简单并列呈现。这种做法在政治表述上或许能够维持表面的安全，但从制度理论的角度审视，却显得不够严谨。

二、立法看似“完整”，核心问题却“悬而未决”

从形式上看，草案结构完整、覆盖面广泛，几乎涵盖了民族团结、文化认同、语言文字、教育、经济发展等所有与民族问题相关的领域。然而，从实质内容来看，这部法律并未正面回应中国民族政策中最为核心的问题之一：民族平等究竟应当以“族群”为中心，还是以“公民个体”为基础？

草案中虽反复重申民族平等、禁止歧视、维护团结等原则，但这些原则早已在宪法以及多部现行法律中有所体现。新法并未在权利配置、制度边界或责任结构等方面提出具有创新性的解决方案，而是对既有的政治结论进行了系统化的重述。

大量条文采用“坚持”“推动”“引导”等表述方式，却缺乏明确的法律要件和可裁判标准，使得这部法律更像是一部政策宣示型的法律，而非能够切实解决现实冲突的规范工具。

三、“共同体意识”抽象空洞，主体文化持续边缘化

草案将“中华民族共同体意识”确立为核心原则，但一个关键问题始终未得到正面回应：这一共同体究竟以何种文化作为公共基础？

从历史事实的角度考察，中华文明并非是由56种文化简单并列拼接而成，而是在漫长的历史演进过程中，以汉文化为主体框架，不断吸收、融合其他民族文化，最终形成了多元一体的文明结构。这一事实并不否认文化的多元性，恰恰相反，多元性正是在主体文化的整合作用下得以实现和发展的。

然而，在当下的制度与话语体系中，对主体民族文化的整合作用往往被刻意弱化，甚至采取回避的态度，仿佛承认这一事实就意味着对其他民族的不尊重。其结果是，“共同体意识”被高度抽象化，缺乏现实的文化载体作为支撑，只能依赖行政手段进行推动，而难以形成自然的认同。

四、从“历史阶段性制度”到“长期治理逻辑”：错位的现实困境

如果承认民族区域自治是一种特定历史阶段的制度安排，那么就必须直面一个现实问题：当国家治理目标已经转向以融合为核心的共同体建设时，这一制度应当如何实现转型？

草案对此选择了回避的态度，既未对自治制度的阶段性特征进行讨论，也未提出向以公民平等为基础的治理逻辑过渡的路径设计。最终导致自治制度在实践过程中不断被去政治化、去权能化，但在话语层面却又持续被强化为“不可动摇的原则”，从而形成一种制度悬置的尴尬状态。

五、结语：民族团结需扎根于坚实的逻辑基础

总体而言，《民族团结进步促进法（草案）》并非是一部通过制度创新来回应现实矛盾的法律，而是一部典型的政治整合型、叙事确认型立法。

它回避了几个关键问题：民族平等与人人平等之间的内在关系、共同体建设与民族自治之间的逻辑冲突、主体文化在多民族国家中的制度角色。而这些被回避的问题，恰恰是决定民族治理能否实现长期自洽的核心要素。

如果民族团结始终停留在对差异的反复强调层面，却无法回归到以宪法为基础的公民平等原则；如果共同体建设无法正视自治制度的阶段性属性，那么即便出台再多的宣示性立法，也只能在文本上呈现出表面的完整，却难以在现实社会中真正发挥实效。

“你们应该多用 Bash。”

过去几周，Anthropic 的 Thariq 和几十家做通用智能体的公司开了电话会议。邮件助手、客服机器人、日程管理——各种产品形态都有。聊完一圈，他发现自己反复在说同一句话。

Bash？那不是程序员用的命令行工具吗，和这些产品有什么关系？

先看一个具体场景。

假设你有一个邮件 Agent，你问它：“这周我在打车上花了多少钱？”

传统做法是这样的：Agent 调用 API 拉取邮件，可能一次性取回 100 封，然后让模型从里面找 Uber、Lyft 的收据，加总金额。

问题在于 100 封邮件塞进上下文，模型要同时记住这些内容，从中筛选、计算。这对大语言模型来说并不轻松。容易漏，容易错，而且你没法验证它到底看了哪些邮件。

这就是典型的模型舒适区问题：数据量不算大到需要专门写程序处理，但又超出了模型一次性硬算的能力范围。夹在中间，很尴尬。

Thariq 的方案是：给 Agent 一个 Bash 工具，让它把中间结果存成文件。

听起来很简单，但背后的逻辑很有意思。

传统的工具调用是这样的流程：

工具 → 模型处理 → 输出结果

所有中间状态都在模型的“脑子”里，你看不见，也没法检查。

换成 Bash 之后，流程变了：

工具 → 存文件 → 搜索/过滤 → 模型处理 → 输出结果

模型可以先把 100 封邮件存到一个文件里，然后用 grep 搜“Uber”，再 grep“Lyft”，分别统计。每一步都有迹可查，最后加总的时候，它还能回头检查自己的中间结果。

这带来三个能力升级：

可复现。同样的命令再跑一遍，结果一样。你可以调试，可以排查问题。

可验证。模型不是凭“记忆”给你答案，而是基于实际文件里的数据。你信不过的话，自己也能打开文件看一眼。

可组合。一个命令的输出可以作为下一个命令的输入，管道一接，复杂任务就能拆成简单步骤。

Bash 让 Agent 从“脑算”变成了“打草稿”。草稿可以留痕，可以检查，可以改。这对需要准确性的任务来说太重要了。

邮件搜索只是最直观的例子。Bash 的能力边界其实很宽。

链式 API 调用是个常见需求。比如“把这周我发过邮件的联系人都找出来”，这需要先拉邮件列表，提取收件人，去重，再逐个查询联系人详情。一连串操作用 Tool calls 来做，调用次数多，中间状态难管理。用 Bash 脚本串起来，逻辑清晰得多。

视频和文件处理也是 Bash 的强项。ffmpeg 这个命令行工具，模型用起来得心应手。找视频里某个片段、裁剪、转码，一行命令搞定。

还有定时任务。在 Agent 运行的容器里，用 cronjob 或 at 命令就能创建定时执行的任务。用户说“每天早上 8 点给我发一份新闻摘要”，Agent 可以自己设好闹钟。

这些场景有个共同点：都需要多步骤操作，都需要保存中间状态，都超出了单次工具调用的能力范围。

但 Bash 是把双刃剑。

能执行命令意味着能做很多事，也意味着能做很多危险的事。rm -rf 一不小心就能删光整个目录。如果 Agent 被恶意提示词攻击，后果可能很严重。

Anthropic 显然考虑到了这一点。他们在 Claude Agent SDK 里做了一套权限系统，包括 Bash 命令解析器和分级权限控制。哪些命令可以直接执行，哪些需要用户确认，哪些完全禁止，都可以配置。

我用 Claude Code 的体会是，这套权限系统确实降低了心理负担。它会在执行敏感操作前询问你，而不是闷头就干。但安全护栏不是万能药。权限系统本身也可能有漏洞，Bash 解析器也可能被绕过。

安全护栏是必需品，但不能因此就觉得万事大吉。

强调 Bash 的好处，也得说清楚它的边界。

如果任务足够简单，别用。“今天天气怎么样”这种一次性查询，直接调 API 返回结果就行，没必要存文件再处理。杀鸡用牛刀反而更慢。

如果环境是 Serverless 的，用不了。很多云函数运行时没有可持久化的文件系统，Bash 的“存中间结果”优势就没了。

如果对安全要求极高，谨慎使用。命令注入的风险无法百分之百消除，金融、医疗这类场景可能更适合用白名单式的专用工具，而非通用的 Bash。

工具的选择取决于场景，而不是工具本身的强弱。Bash 很强，但不是所有场合都该用。

回过头看，Thariq 这条建议的真正价值不是“Bash 很强”这个结论，而是背后的思维方式：

让 Agent 的思考过程“落地”到可检查的中间产物。

传统的 Agent 设计把所有东西都塞进模型的上下文，一锤子买卖。Bash 提供了另一种路径：把复杂任务拆开，每一步都留下痕迹，可以验证，可以回溯。

想想看，这和人类处理复杂问题的方式多像。我们做复杂计算时会列竖式，写长文章时会先拟提纲，处理大量信息时会做笔记。不是因为脑子记不住，而是因为落到纸上更可靠、更容易检查。

Agent 也一样。不是说模型处理不了，而是有中间产物的流程更值得信任。我自己用 Agent 辅助写作，所有中间产物都会存成文件：网络检索资料、提纲、不同版本的草稿、画图的提示词。这些存下来后续就可以灵活组合。

Bash 不只是程序员的工具，更是让 Agent 具备可验证、可复现、可审计能力的关键一环。

在前现代社会，国家对老百姓一无所知——不知道你有多少财产、住在哪里、甚至不知道你是谁。

所以国家建立了"让人可见"的基础设施：人口普查、姓氏、地图。

这叫"legibility"（可读性）。可读性是统治的前提。

后来，企业也学会了这一套。它们收集你的行为数据，但你无法访问、更无法理解这些数据。

你是历史上被测量得最多的人，但对自己最不透明。

她的观点是：Claude Code让这种权力关系逆转了。

以前是国家/企业监控你。现在你可以监控自己。

监控塔还在，但塔属于你。

几个值得借鉴的点

  1. 从"单点使用"到"系统化架构"

大多数人用Claude Code的方式是：有任务了，开个session，搞定，关掉。

Molly的做法是：8个目录，8个独立的agent，24小时运行，各司其职。

这是两种完全不同的思路。前者是"工具"，后者是"基础设施"。

  1. 从"AI帮我做事"到"AI帮我看见自己"

这个视角转换很有意思。

传统思路：AI是执行者，帮你完成任务。

她的思路：AI是观察者，帮你汇总所有信息流，识别模式，生成行动建议。

这不只是效率工具，是一种新的自我认知方式。

  1. 保持在系统之外的meta层

当你的生活完全量化、完全依赖系统，你可能会失去一些东西。

Goodhart定律：当你优化一个指标，指标本身可能会失效。

她的做法是：当WHOOP说"恢复良好"但感觉像死了一样，她会注意到。当~/trades的论点错了，她会亏钱。

这种"活在系统之外"的能力很重要。当指标变成游戏，你要能识别出来。当系统不再服务于你，你要能删掉它。

怎么开始？

如果你也想尝试类似的架构，我的建议是：

第一步：从一个领域开始

不要一上来就搞8个并行实例。

先选一个你最需要的领域：邮件、写作、投资、健康，随便哪个。

搭建一个Claude Code工作流，跑通它。

第二步：结构化你的目录

~/claude/

├── work/ # 工作相关

├── personal/ # 个人事务

├── finance/ # 财务/投资

└── health/ # 健康

每个目录有自己的

，定义规则和工作流。

第三步：考虑自动化

Molly用的是cron job + caffeinate + 短信通知。

你也可以用GitHub Actions做定时触发，或者用Mac的launchd来跑定时任务，再配合各种webhook和API集成。

第四步：记录和迭代

她说"所有思维轨迹都被记录，用于递归自我改进"。

这个很重要。让你的agent学习自己的错误。

最后

Karpathy说过一句话，大意是：现在还不用AI编程工具，那是你自己的问题。

Molly Cantillon可能是我见过的把这个理念执行得最彻底的人。

她不是把AI当工具。她是把AI当基础设施，把整个生活建构在上面。

几百年来，可读性（legibility）的流向是向上的：你是被监控的对象，机构是监控者。

现在，方向可以反过来了。

管理好你自己。

宋江老师为什么混的不如高俅老师好呢？网上都说因为宋江是小地方人，高俅是“北京人”，是京爷，见多识广，有这方面的原因，但是关键不在这里。

首先你要明白，宋江这种人，他能力强，是什么能力强？

宋江这种人，能力有短板，而且不稀缺。其实你去中国任何一个县城，都能找到一个或几个宋江，中国的每个县城都有几个人脉特别广办事能力特别强的人物，这种人你乍一见面，会觉得很可惜，觉得他的能力和地位是完全不匹配的，这么强的能力怎么屈居一个县城，但是你见这种人见得多了就会发现这种人，其实并没有初见时想象得那么厉害。

比如宋江这个人，他有一个很明显的能力短板，他搞定人的能力很强，但是他搞不定女人，他既搞不定阎婆惜，碰见李师师这种高端场合也会局促，搞不定女人，其实说明他的很多问题，这就是你看见的所有这种宋江式的人物的缺陷，我们看到的所有宋江的魅力，你会发现，都是来自于「下级」，无论是纳头便拜的江湖兄弟，还是战败的降将也好，宋江处于的生态位，一直是一个大哥的生态位，但是对上的能力，宋江只有一条，服从，无限服从，这当然说明宋江的执行力很强，但是同时也说明一点，宋江应对上级，也没有多少办法。

这跟宋江的底色有关系。

宋江混的不如高俅，根本原因就是他们的底色不同。宋江的底色是做题家，他作为一个押司，基本掌管了整个郓城县的刑狱，所以他为什么作为一个小吏能成为一个大地皮蛇，所有的好汉都来投奔，因为他有一个官方的身份。现在在网络上，随便一个帽子叔叔，都有很多粉丝追捧，帽子叔叔在体制内其实没有什么身份，甚至算公务员鄙视链的中低层，但是在互联网上却能被捧得很高。

为啥呢？因为帽子叔叔就是普通人认知，权力的直接化身，普通公务员同时又掌握暴力的具体模板。

宋江的能力没得说，很厉害，也有正式编制，只不过因为宋朝的体制问题很难转成官员，除非运气爆棚，立有军功，或者有力的上级举荐，才有机会转为官员，当然也有考试，部分吏员可通过“明法科”等专业考试获得入仕资格，但是一辈子都是技术干部，跟正经科举所谓正途上来的完全不一样，所以给他的上升路径比较狭窄。这也是宋江一生的不得志处，因为成为一个好汉、成为一个首领不是他想要的，他想要的就是光宗耀祖、光耀门楣，要的是科举得中，而他并不是一个成功的做题家。

高俅的底色是运动明星，他的大部分时间是在教人使钱，替京城的富家子弟们花钱。所以他整个发家的路线，是从一个富裕的商人家庭（很可能是做上层生意的）再进入一个官僚家庭，从苏学士再到端王。然后端王发现他的才华就是伺候人，是叠码仔，交际花，是一个帮闲的角色。

宋江和高俅混他们俩一个是帮忙的，一个是帮闲的。鲁迅先生讲过有两种人一种人是帮忙的人，一种人是帮闲的人，对于端王也就是宋徽宗他们这些上层的人来说，表面上喜欢帮忙的人，实际上喜欢的是帮闲的人，因为他们本质上都是吃喝玩乐的纨绔子弟。

宋江最大的问题就是他没有玩过，别看他在郓城县也是有数的富户，是宋庄主，但是他依然属于本本分分的良家子，宋江对女人完全没有办法，对于高端场合的娱乐完全不熟悉。

宋江想去东京找一下李师师搭线，他带的是柴进和燕青，柴进是前朝贵族，而燕青是卢俊义的手下，卢俊义是北京（大名府）首富，是帮卢俊义使钱社交的，同样也是体育小明星，因为卢俊义跟宋江也是一种人，除了正事和武术，对别的不感兴趣。

燕青就是长得好看的帮闲，柴进天生贵气，燕青小意琢磨，察言观色的能力非常强。其实燕青和高俅本质上是一样的，高俅擅长的是蹴鞠，而燕青对于什么双陆棋子、弹弓之类的东西无所不通，他们都有和上层社交的工具。

但宋江不行，宋江在李师师眼里就是一个土头土脑的土豪大哥，是乡下的地主。你可以说宋江非常有本事、有能力，有定国安邦的才能，但是他往上走的时候这些大家都看不见，因为他缺少社交名片。

宋江混的不如高俅的第二个原因还是由第一个原因造成的，宋江没有对上的能力，宋江所有的本事、智慧全部来自对下，是一种衙门智慧。

宋江在前期都是被捧的那一个，好汉、郓城县的人对他都是拜大哥。何涛十万火急来郓城县，却要先等宋江办完自己事，不敢逾越雷池一步，足以说明宋江在郓城县是个地头蛇的地位。

高俅一直在向上社交，而宋江在向下兼容。宋江的向上社交的能力都来自上面人对他的欣赏，而不是他主动地逢迎谄媚，央视的水浒可能给大家一个错误的印象，宋江给宋徽宗下跪，他跪在金銮殿上屁股撅的比山还高，这是一个非常有意象的转变，象征他对皇权的匍匐，但这其实不代表他是一个会谄媚的人。

宋江的智慧是介于创业智慧和庙堂智慧之间的，他的智慧是定国安邦，是正路的智慧，是办事的智慧。而高俅是街头智慧，和街头混混没有区别，对上逢迎拍马，对下欺凌辱骂。宋江对下是非常宽厚的，他很擅长拉拢众人的心。而高俅当不了领导，他手底下的有才之士比如王进林冲都被他赶跑了，而对宋徽宗，他确实是宋徽宗心腹，但是他对宋徽宗敢于欺瞒，你试想一下，宋江放到高俅的位置上，且不说，他没有踢球的本事，即使有，他也会劝谏宋徽宗，他也不会去欺瞒上级，因为宋江这个人表面厚黑，其实本质上是要做事的，是要辅佐上司做贤君圣主的。

但是很可惜，不是所有皇帝都想成为贤君圣主，至少宋徽宗不是。宋江混的不如高俅的第三个原因就是宋徽宗的问题，也是水浒里书写的当时宋朝的官僚问题。因为当时的天下，高层都喜欢具有街头智慧的人，庙堂本来至少应该是一个衙门，实际上却变成一个佞臣的天下，一个帮闲的天下，不但不如高级衙门，甚至不如低级衙门。因为宋徽宗就喜欢带他踢球、给他搞花石纲的人。如果宋江碰到的是李世民、赵匡胤这种实实在在的明君，他肯定平步青云，因为这些人都是干事儿的皇帝，他们永远缺帮忙的人。但是宋徽宗时期天下承平已久，他自以为已经很好了，觉得自己在盛世。所以他就需要街头智慧，宋江倒也不是没有街头智慧，但是他是不愿意拿街头智慧，去迎合上司的。

第四个原因是路线的问题，宋江也试图走高俅、柴进的路线，但是他走不通，所以只能走宿元景的路线，宿元景在书里是一个清廉形象，但是这派的路线梯队能够给到宋江的资源非常有限，因为宿元景已经属于朝堂的边缘，宋江只能是边缘的边缘。所以混得好不好一定和你所在的梯队非常相关，人一定要上面有人，宋江往上走到天上，能摸到的人就是宿元景，而高俅的顶端能摸到的是宋徽宗。
